34 /*
35  * A singly-linked list is headed by a single forward pointer. The
36  * elements are singly linked for minimum space and pointer manipulation
37  * overhead at the expense of O(n) removal for arbitrary elements. New
38  * elements can be added to the list after an existing element or at the
39  * head of the list. Elements being removed from the head of the list
40  * should use the explicit macro for this purpose for optimum
41  * efficiency. A singly-linked list may only be traversed in the forward
42  * direction. Singly-linked lists are ideal for applications with large
43  * datasets and few or no removals or for implementing a LIFO queue.
44  */
45

212 /* A circle queue is headed by a pair of pointers, one to the head of the
213  * list and the other to the tail of the list. The elements are doubly
214  * linked so that an arbitrary element can be removed without a need to
215  * traverse the list. New elements can be added to the list before or after
216  * an existing element, at the head of the list, or at the end of the list.
217  * A circle queue may be traversed in either direction, but has a more
218  * complex end of list detection.
219  */
220
