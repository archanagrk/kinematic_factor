2 /* $Id: basic_xpath_reader.h,v 1.18 2007-07-17 16:56:10 bjoo Exp $
3  *
4  * File: basic_xpath_reader.h
5  *
6  * Description:
7  *
8  * This file specifies the basic functionality of the reader.
9  *
10  * These consist of
11  * i) Opening files
12  * ii) Getting at primitive types (non-templated types)
13  * iii) Getting at attributes
14  * iv) Executing arbitrary XPATH queries
15  * v) Displaying the results of the Xpath Queries
16  * vi) Cleaning up
17  *
18  * Despite the length of this file there are only a few functions
19  * you need to worry about here, since most of the file deals with
20  * overloading things.
21  *
22  * Functionality:
23  *
24  * open -- from file or istream: opens a document
25  * close -- cleans up
26  *
27  * evaluateXPath(query string) -- runs the XPATH query
28  * the query result is returned in the internal variable
29  * query result which is an xmlXPathObjectPtr. If you want to
30  * mess with this you have to get down to the level of libxml2
31  *
32  * printQueryResult(ostream) -- print the results of the query to
33  * the ostream supplied. Useful if you
34  * need the result tagset of the query as
35  * a string, or for debugging or whatever.
36  *
37  * count(query_string) -- runs the xpath query "count(query string)"
38  * and returns its value.
39  *
40  * AND THE MOST IMPORTANT FOR LAST:
41  *
42  * get(xpath_query_string, <result>) -- Executes the XPath Query
43  * and stores the result in the variable
44  * result. This is a highly overloaded function
45  * and can deal with most language basic types
46  * (int, unsigned int,
47  * short, unsigned short,
48  * long, unsigned long,
49  * float, double,
50  * bool, std::string)
51  *
52  * The semantics are:
53  * - the query string must identify a unique element node with simple
54  * content
55  *
56  * - only 1 occurrance of the desired object can occur in the
57  * identified tag. I.e: sequences of object within type eg <foo> 2 3
58  * </foo> are not allowed. In the case of string, the tag can contain
59  * and arbitrary string.
60  *
61  * - internalisation is LOST. xmlChar -s are converted to char * and
62  * then to std:string
63  *
64  * - boolean values have to be "true" or "false" rather than 0 or 1.
65  *
66  * Given that the above are satisfied failure can still occur if
67  * the >> operator for the type cannot convert from the string of the
68  * type to the desired type.
69  *
70  *
71  * getAttribute(xpath_query_to_node, attribute_name, <result>)
72  *
73  * This is much like get, except it tries to get attributes attached
74  * to a given element node. The same amount of overloading is done
75  * as for get.
76  *
77  * Semantics are:
78  *
79  * - the xpath_query_to_node must identify a unique element node
80  * which has the desired attribute.
81  *
82  * - the semantics of the form of the attribute are the same as for
83  * get()
84  *
85  *
86  * ERRORS ETC:
87  * ===========
88  *
89  * If the semantics are not met, or queries fail, C++ exceptions will be
90  * thrown. These need to be caught to stop your program from "Abort" ing.
91  * The type of exceptions thrown in this library is always const string&
92  * so that you can immediately dump the error message to wherever you like
93  *
94  * Have fun,
95  * Balint
96  */
