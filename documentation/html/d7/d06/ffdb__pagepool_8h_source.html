142 /*
143  * The memory pool scheme is a simple one. Each in-memory page is referenced
144  * by a bucket which is threaded in up to two of three ways. All active pages
145  * are threaded on a hash chain (hashed by page number) and an lru chain.
146  * Inactive pages are threaded on a free chain. Each reference to a memory
147  * pool is handed an opaque MPOOL cookie which stores all of this information.
148  */
149 #define FFDB_HASHSIZE 16384
